# ActivityHub Backend API 

Backend-часть высоконагруженного мобильного приложения для персонализированного подбора досуга. Проект разработан в рамках курсового проектирования в Университете ИТМО.

## Основная цель
Создание отказоустойчивого и безопасного API, способного обрабатывать запросы от мобильных платформ (Android/Kotlin) с поддержкой интеллектуальной фильтрации контента и многофакторной аутентификации.

## Технологический стек
*   **Язык:** Python 3.12 (Asyncio)
*   **Фреймворк:** FastAPI (Stateless REST API)
*   **База данных:** PostgreSQL 15 (Supabase)
*   **ORM:** SQLAlchemy 2.0 (Асинхронный движок)
*   **Безопасность:** JWT, Bcrypt, 2FA (Email SMTP)
*   **Инфраструктура:** Docker, Supavisor (Connection Pooling), Ngrok

## Ключевые особенности и задачи
- **Безопасность (Security-first):**
    - Реализована система регистрации с подтверждением через email.
    - Внедрена двухфакторная аутентификация (2FA) для защиты пользовательских данных.
    - Хеширование паролей с использованием алгоритма bcrypt.
- **Производительность:**
    - Полностью асинхронная архитектура (от обработки запроса до запроса в БД).
    - Оптимизация времени отклика (Latency) до < 200 мс при нагрузке в 1000 одновременных пользователей.
- **Гибкая схема данных:**
    - Использование типа JSONB для хранения настроек и предпочтений пользователя, что позволяет расширять функционал без изменения структуры таблиц (NoSQL-подход внутри SQL).

## Решение технических вызовов (Case Study)
В процессе разработки была решена проблема сетевой связности IPv6. При подключении к облачной БД Supabase из сетей некоторых российских провайдеров возникала ошибка OSError: [Errno 101] Network is unreachable.
**Решение:** Архитектура была перенастроена на использование Transaction Connection Pooler (Supavisor) на порту 6543, что обеспечило 100% доступность сервиса вне зависимости от типа сетевого окружения.

## Запуск проекта

### 1. Подготовка окружения
Создайте файл .env в корневой директории, используя образец из env_example.

### 2. Запуск через Docker (рекомендуется)
docker build -t activityhub-backend .
docker run -p 8000:8000 --env-file .env activityhub-backend

### 3. Локальный запуск
# Установка зависимостей
pip install -r requirements.txt

# Запуск сервера
uvicorn App.main:app --host 127.0.0.1 --port 8000 --reload
